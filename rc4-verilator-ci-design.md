# rc4 recommendation

Ship rc4 with the staged-rollout architecture at ratchet mode: ubuntu-only, 7-way directory-sharded MatX-verilator leg (pinned build-20260701_1 + python3.11 shim) whose sole gate is an if:always() verdict job diffing observed results against committed manifests seeded from local baseline3 — red only on NEW failures, UNSUPPORTED-set drift, or a dead/incomplete shard, never on the catalogued tail or on burn-down fixes landing in parallel (stale entries warn, they don't red). Do not run the committed 81a91b38 single-job boolean gate (it would be red today, and its single-job -j4 form has a 4/4 runner-death record), do not flip it to continue-on-error (the PR-1004 hollow-green anti-pattern), and do not hold the release for the full burn-down (the tail is open-ended triage that the manifest already pins). Turn the macOS verilator gating leg off (linux-x86_64 is what MatX deploys; 3-core/7GB OOM risk and observed 24h queue starvation would hold the release hostage). Sequence: land the uncommitted verinorun skip layer + machinery, run local baseline3 at -j24 to seed manifests, push the branch, one observe-mode shakedown dispatch (~1.5h — the first-ever hosted-runner completion proof, ccache warmer, and CI-vs-local manifest reconciliation), commit the reconciled manifests, then dispatch the real release at ratchet and feed the green run id to matx-prerelease. If the shakedown itself fails on infrastructure, fall back to publishing at the rc3 iverilog-green standard with the observe ledger recorded — never by suppressing a red gate. Total cost: roughly one working day, two dispatches.

Proves the one thing never proven — 5.x suite completion on hosted runners — plus shard balance, warms per-shard ccache (save-on-red split makes warmth reachable even if red), and surfaces the CI-vs-local manifest delta (glibc/g++/4-core timing). Reconcile deltas into fails/flaky manifests; commit. If a shard starves at -j4: drop to -j3, rerun observe (escalation ladder; swap already in place).
Stage 2 (rc4 release): dispatch with defaults (ratchet). Green = the release regresses nothing vs the fully-catalogued deployed-verilator state, every deviation is an enumerated dispositioned manifest entry, and the UNSUPPORTED contract holds. Feed the run id to matx-prerelease. SHIP.
